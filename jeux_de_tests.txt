CHECKLIST DE VALIDATION — À VÉRIFIER POUR CHAQUE CAS
═══════════════════════════════════════════════════════════════════════════════

Pour chaque scénario, cochez les points suivants :

ÉCRAN 2 — Questions
  □ 5 questions posées séquentiellement (Q1 → Q2 → Q3 → Q4 → Q5)
  □ Chaque réponse acceptée et enregistrée dans l'historique
  □ Passage automatique à l'écran 3 après Q5

ÉCRAN 3 — Synthèse IA + Revue médecin
  □ diagnostic_summary affiché (généré par DiagnosticAgent + LLM)
  □ interim_care affiché (généré par MCP recommend_interim_care)
  □ Score de gravité cohérent avec le cas
  □ Champs médecin remplissables et soumis

ÉCRAN 4 — Rapport final
  □ Nom patient affiché
  □ Motif de consultation affiché
  □ Historique Q/R visible (expandable)
  □ Synthèse diagnostique IA présente
  □ Recommandations MCP présentes
  □ Observations médecin présentes
  □ Plan de traitement présent
  □ Avertissement académique affiché

API — Endpoints (tester via http://localhost:8000/docs)
  □ POST /start_session        → session_id retourné
  □ POST /start_consultation   → thread_id + première question
  □ POST /consultation/resume  → question suivante ou status=physician_review
  □ GET  /consultation/{id}    → status + diagnostic_summary + interim_care
  □ GET  /consultation/{id}/report → rapport complet
